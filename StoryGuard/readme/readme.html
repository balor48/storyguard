1. Introduction

Welcome to Story Database, a comprehensive web application designed to help writers organize and manage all aspects of their stories. Whether you're working on a novel, screenplay, or game narrative, Story Database provides the tools you need to create detailed outlines, manage characters and locations, map relationships, and visualize your story's timeline.

This documentation provides a detailed guide to using Story Database, including step-by-step instructions, screenshots, and tips to help you get the most out of the application.

Key Features

Character Management

Create and manage detailed character profiles, including personal information, story roles, relationships, and custom fields.

Location Management

Define and organize locations within your story world, including descriptions, images, and related elements.

Relationship Mapping

Visualize and manage relationships between characters with an interactive network diagram.

Plot Management

Outline plot points, story arcs, and chapters, linking them to characters and locations.

World-Building

Create and manage world-building elements, such as cultures, histories, and magic systems.

Timeline Visualization

Display story events in chronological order to maintain consistency and track character arcs.

Statistics and Analysis

Generate statistics and insights about your story elements to identify patterns and gaps.

Cloud Storage Integration

Integrate with cloud storage providers for data backup and synchronization across devices.

2. Getting Started

System Requirements

Story Database is a web-based application that runs in your browser. For optimal performance, we recommend:

  • A modern web browser (Chrome, Firefox, Edge, or Safari)
  • JavaScript enabled
  • At least 4GB of RAM
  • Screen resolution of 1280Ã—720 or higher

First-Time Setup

When you first open Story Database, you'll be presented with the main interface. Here's what you should do to get started:

  1. Navigate to the Dashboard: Click on the "Dashboard" tab in the navigation bar to see an overview of your story elements.
  2. Create Your First Character: Click on the "Characters" tab and use the form to add your first character.
  3. Add a Location: Navigate to the "Locations" tab to add your first story location.
  4. Define Relationships: Once you have multiple characters, you can define relationships between them in the "Relationships" tab.
  5. Add Plot Points: Use the "Plots" tab to start outlining your story's plot points.
  6. Build Your World: Add world-building elements in the "World-Building" tab.
Tip: Start with the core elements of your story (main characters, key locations, and major plot points) before adding more detailed information. This will help you establish a solid foundation for your story database.

Data Storage

By default, Story Database stores all your data in your browser's local storage. This means:

  • Your data remains on your computer
  • No internet connection is required to use the application
  • Your data persists between browser sessions
Important: Local storage has limitations. We strongly recommend regularly backing up your data using the export functions or setting up cloud storage integration to prevent data loss.

4. Character Management

The Characters tab allows you to create and manage detailed character profiles for your story.

The Characters tab provides tools for creating and managing character profiles.

Character Form

The character form is divided into three main sections:

Personal Information

  • Image Upload: Add a character portrait or reference image
  • Title: Character's title (e.g., Dr., King, Captain)
  • First Name: Character's first name (required)
  • Last Name: Character's last name
  • Name Generator: Generate random names based on race
  • Sex: Character's sex
  • Race: Character's race or species

Story Information

  • Role: Character's role in the story (e.g., Protagonist, Antagonist, Mentor)
  • Series: The series this character belongs to
  • Book: The book this character appears in
  • Character Relationships: Define relationships with other characters
  • Tags: Add tags to categorize and filter characters

Additional Details

  • Custom Fields: Add custom fields for additional character information
  • Notes: Rich text editor for detailed character notes

Adding a New Character

  1. Fill out the character form with the character's information
  2. Click the "Add" button to add the character to your database

Character Table

The character table displays all characters in your database with the following columns:

  • Image: Character's image (if uploaded)
  • Title: Character's title
  • First Name: Character's first name
  • Last Name: Character's last name
  • Sex: Character's sex
  • Race: Character's race
  • Series: The series this character belongs to
  • Book: The book this character appears in
  • Notes: Preview of character notes
  • Actions: Edit, delete, and view options

Searching and Filtering Characters

Use the search bar at the top of the character table to search for characters by name, series, book, etc. You can also use the advanced search options to filter characters by specific criteria.

Tag Cloud

The tag cloud displays all tags used for characters, with the size of each tag indicating how frequently it's used. Click on a tag to filter characters by that tag.

Editing a Character

  1. Click the "Edit" button in the Actions column for the character you want to edit
  2. The character form will be populated with the character's information
  3. Make your changes
  4. Click the "Update" button to save your changes

Deleting a Character

  1. Click the "Delete" button in the Actions column for the character you want to delete
  2. Confirm the deletion when prompted
Warning: Deleting a character will also delete all relationships associated with that character. This action cannot be undone.

Viewing Character Details

  1. Click the "View" button in the Actions column for the character you want to view
  2. A modal will open displaying all details for the character
Tip: Use the name generator to quickly create names for minor characters or when you're stuck for ideas. You can generate names for different races, such as Human, Elf, Dwarf, etc.

6. Relationship Mapping

The Relationships tab allows you to visualize and manage relationships between characters in your story.

Relationship Network Diagram

The relationship network diagram provides a visual representation of character relationships. Each character is represented by a node, and relationships between characters are represented by lines connecting the nodes.

Filtering Relationships

Use the filters at the top of the Relationships tab to focus on specific types of relationships or characters:

  • Relationship Type: Filter by relationship type (e.g., Friends, Family, Allies)
  • Character: Filter to show only relationships involving a specific character

Adding Relationships

Relationships are added through the Character form in the Characters tab:

  1. Navigate to the Characters tab
  2. Edit an existing character or create a new one
  3. In the "Character Relationships" section, select another character and the relationship type
  4. Click the "Add" button to create the relationship

Relationship Types

Story Database supports the following relationship types:

  • Friend: Characters who are friends
  • Family: Characters who are related by blood or marriage
  • Ally: Characters who work together toward a common goal
  • Enemy: Characters who are in conflict with each other
  • Mentor: Character who guides or teaches another character
  • Student: Character who learns from another character
  • Lover: Characters in a romantic relationship
  • Rival: Characters who compete with each other
  • Other: Any other type of relationship

Relationship List

Below the network diagram, you'll find a list of all relationships in your database. Each entry includes:

  • The names of the two characters
  • The relationship type
  • Options to edit or delete the relationship

Editing a Relationship

  1. Click the "Edit" button next to the relationship you want to edit
  2. Select a new relationship type from the dropdown menu
  3. Click "Save" to update the relationship

Deleting a Relationship

  1. Click the "Delete" button next to the relationship you want to delete
  2. Confirm the deletion when prompted
Tip: Use the relationship network diagram to identify isolated characters (those with few or no connections) or to visualize character groups and factions within your story.

9. Timeline Visualization

The Timeline tab allows you to visualize your story's events in chronological order, helping you maintain consistency and track character arcs.

Timeline Filters

Use the filters at the top of the Timeline tab to focus on specific aspects of your timeline:

  • Series: Filter by series
  • Book: Filter by book

Timeline Display

The timeline displays events in chronological order, with each event represented by a card containing:

  • Event title
  • Event date/time
  • Characters involved
  • Location
  • Brief description

Adding Events to the Timeline

Events are automatically added to the timeline based on plot points you create in the Plots tab. To add an event to the timeline:

  1. Navigate to the Plots tab
  2. Create a new plot point or edit an existing one
  3. Ensure the plot point has an Order/Sequence value to determine its position in the timeline
  4. The plot point will appear in the timeline based on its sequence value
Tip: Use the timeline to identify potential inconsistencies in your story's chronology. For example, if a character appears in two different locations at the same time, the timeline will help you spot this issue.

11. Data Management

Story Database provides several options for managing your data, including import, export, and backup functions.

Exporting Data

You can export your data in JSON format for backup or transfer to another device:

  1. Navigate to the tab containing the data you want to export (Characters, Locations, etc.)
  2. Click the "Export" button in the file controls section
  3. Choose a location to save the exported file
  4. Click "Save"

Exporting to PDF

You can export your data to PDF format for printing or sharing:

  1. Navigate to the tab containing the data you want to export (Characters, Locations, etc.)
  2. Click the "PDF" button in the file controls section
  3. A PDF will be generated and downloaded automatically

Importing Data

You can import data from a previously exported JSON file:

  1. Navigate to the tab where you want to import data (Characters, Locations, etc.)
  2. Click the "Import" button in the file controls section
  3. Select the JSON file you want to import
  4. Click "Open"
  5. Confirm the import when prompted
Warning: Importing data will overwrite any existing data of the same type. Make sure to back up your current data before importing.

Backing Up Data

You can create a backup of your data that will be stored in your browser's local storage:

  1. Navigate to the tab containing the data you want to back up (Characters, Locations, etc.)
  2. Click the "Backup" button in the file controls section
  3. Enter a name for the backup when prompted
  4. Click "OK"

Restoring from Backup

You can restore your data from a previously created backup:

  1. Navigate to the tab containing the data you want to restore (Characters, Locations, etc.)
  2. Click the "Backup" button in the file controls section
  3. Select "Restore from Backup" from the dropdown menu
  4. Select the backup you want to restore from the list
  5. Click "Restore"
  6. Confirm the restoration when prompted
Tip: Create regular backups of your data to prevent loss due to browser cache clearing or other issues. We recommend exporting your data to JSON files as an additional precaution.

12. Book Analysis

The Book Analysis feature allows you to analyze existing manuscripts to extract character information, helping you quickly populate your character database from your written work.

The Book Analysis tab provides tools for analyzing manuscripts and extracting character information.

Accessing Book Analysis

To access the Book Analysis feature:

  1. Click on the "Analyze Book" tab in the navigation bar (or use the Alt+9 keyboard shortcut)
  2. The Book Analysis interface will load, displaying analysis options and a file upload area

Analysis Options

The Book Analysis feature offers several options for analyzing your manuscript:

Minimum Character Mentions

Set the minimum number of times a name must appear in the text to be considered a character. This helps filter out names that are mentioned only once or twice.

Detection Methods

  • Dialogue Attribution: Detects character names in dialogue attribution (e.g., "said John")
  • Named Entity Recognition: Uses natural language processing to identify names of people
  • Capitalized Word Analysis: Identifies capitalized words that might be names
  • Frequency Analysis: Analyzes word frequency to identify potential character names
  • Title & Honorific Detection: Identifies titles and honorifics (e.g., Mr., Dr., King)
  • Direct Address Pattern: Detects names in direct address (e.g., "Hello, John")
  • Possessive Form Detection: Identifies possessive forms of names (e.g., "John's")
  • Character Introduction: Detects character introductions (e.g., "a man named John")

Additional Options

  • Combine name variants: Combines different forms of the same name (e.g., "John" and "Johnny")
  • Filter common words: Filters out common words that might be mistaken for names

Uploading a Book

To upload a book for analysis:

  1. In the Book Analysis tab, locate the "Upload Book" section
  2. Either drag and drop your book file into the designated area or click the "Choose Files" button to select a file
  3. Supported file formats include TXT, DOCX, RTF, and HTML

Analyzing a Book

To analyze a book:

  1. Upload your book file as described above
  2. Configure the analysis options according to your preferences
  3. Click the "Analyze Book" button
  4. The analysis will begin, and a progress indicator will show the status
  5. Once complete, the results will be displayed in the "Character Analysis Results" section

Character Analysis Results

The Character Analysis Results section displays a table of detected characters with the following information:

  • Select: Checkbox to select characters for adding to your database
  • Title: Detected title (if any)
  • First Name: Detected first name
  • Last Name: Detected last name
  • Series: Series selection dropdown (optional)
  • Book: Book selection dropdown (optional)
  • Mentions: Number of times the character is mentioned in the text

Adding Characters to Your Database

To add detected characters to your database:

  1. Select the characters you want to add by checking the boxes in the "Select" column
  2. Optionally, select a series and book for the characters using the dropdown menus
  3. Click the "Add Selected to Database" button
  4. The selected characters will be added to your character database

Generating a PDF Report

To generate a PDF report of the analysis results:

  1. After analyzing a book, click the "Generate PDF Report" button
  2. A PDF will be generated and downloaded automatically
  3. The report includes all detected characters and their mention counts
Tip: For best results, start with the default analysis options and then adjust them based on your specific manuscript. Different writing styles may require different detection methods.

13. Cloud Storage Integration

Story Database can integrate with cloud storage providers to automatically back up and synchronize your data across devices.

Supported Cloud Storage Providers

  • Google Drive
  • Dropbox
  • OneDrive

Setting Up Cloud Storage

  1. Click the "Cloud Settings" button in the file controls section of any tab
  2. Select your preferred cloud storage provider from the dropdown menu
  3. Click the "Connect" button
  4. Follow the authentication process for your selected provider
  5. Once authenticated, select a folder for storing your Story Database files
  6. Click "Save Settings"

Cloud Sync Status

The cloud sync status indicator shows the current state of your cloud synchronization:

  • Green: Data is synced and up to date
  • Yellow: Sync in progress
  • Red: Sync error or not connected

Manual Sync

You can manually trigger a sync with your cloud storage:

  1. Click the "Cloud Settings" button in the file controls section
  2. Click the "Sync Now" button

Disconnecting Cloud Storage

  1. Click the "Cloud Settings" button in the file controls section
  2. Click the "Disconnect" button
  3. Confirm the disconnection when prompted
Warning: Disconnecting cloud storage will not delete your data from the cloud, but it will stop automatic synchronization. Your data will remain in your browser's local storage.
Tip: Cloud storage integration allows you to access your Story Database from multiple devices. For example, you can work on your story on your desktop computer and then continue on your laptop without having to manually transfer files.

14. Tips and Best Practices

Getting the Most Out of Story Database

Start with the Basics

Begin by adding your main characters, key locations, and major plot points. This will give you a solid foundation to build upon.

Use Tags Effectively

Tags are a powerful way to categorize and filter your story elements. Create a consistent tagging system to help you quickly find related elements.

Leverage Relationships

Character relationships add depth to your story. Use the Relationships tab to visualize how your characters interact and identify opportunities for conflict or alliance.

Maintain Chronology

Use the Order/Sequence field for plot points to maintain a clear chronology. This will help you visualize your story's timeline and ensure events occur in a logical order.

Regular Backups

Regularly export your data and/or set up cloud storage integration to prevent data loss. Local storage has limitations and can be cleared by browser updates or cache clearing.

Use Custom Fields

Take advantage of custom fields in the Character form to add information specific to your story's needs. For example, you might add fields for magical abilities, equipment, or psychological traits.

Consistent Naming

Establish consistent naming conventions for series, books, character roles, location types, etc. This will make it easier to filter and organize your data.

Regular Reviews

Periodically review your story elements to ensure consistency and identify areas for development. The Statistics tab can help you identify patterns and gaps.

Use Rich Text Formatting

Take advantage of the rich text editor for notes and descriptions. Use formatting options like bold, italic, and lists to make your content more readable and organized.

Collaborative Writing

If you're collaborating with other writers, use the export/import functions to share your story database. Each writer can work on their own copy and then merge changes.

Tip: Story Database is a tool to support your writing process, not replace it. Use it to organize your ideas and maintain consistency, but don't let it constrain your creativity.

15. Troubleshooting

Common Issues and Solutions

Data Loss

Issue: My data disappeared after clearing my browser cache or using a different browser.

Solution: Story Database uses local storage by default, which is browser-specific and can be cleared. To prevent data loss:

  • Regularly export your data to JSON files
  • Set up cloud storage integration
  • Create regular backups using the Backup function

Performance Issues

Issue: The application is running slowly or becoming unresponsive.

Solution: Performance issues can occur when you have a large amount of data. Try the following:

  • Clear your browser cache and reload the application
  • Export your data, clear local storage, and then import your data again
  • If you have a very large database, consider splitting it into multiple databases (e.g., one per series)

Import Errors

Issue: I'm getting errors when trying to import data.

Solution: Import errors usually occur when the imported file is not in the correct format. Ensure that:

  • The file was exported from Story Database
  • The file has not been modified since export
  • You're importing to the correct tab (e.g., Character data to the Characters tab)

Cloud Storage Sync Issues

Issue: My data is not syncing with cloud storage.

Solution: Cloud storage sync issues can have several causes:

  • Check your internet connection
  • Ensure you're still authenticated with your cloud storage provider
  • Try disconnecting and reconnecting your cloud storage
  • Check if your cloud storage quota is full

Browser Compatibility

Issue: Some features are not working in my browser.

Solution: Story Database is designed to work with modern browsers. If you're experiencing issues:

  • Update your browser to the latest version
  • Try using a different browser (Chrome, Firefox, Edge, or Safari)
  • Ensure JavaScript is enabled in your browser settings

Image Upload Issues

Issue: I can't upload images or images are not displaying.

Solution: Image issues can be caused by:

  • File size too large (try using a smaller image)
  • Unsupported file format (use JPG, PNG, or GIF)
  • Browser storage limitations (clear some space in your browser storage)
Note: If you continue to experience issues that aren't resolved by these troubleshooting steps, try clearing your browser cache and reloading the application. If problems persist, you may need to export your data, clear your browser's local storage, and then import your data again.
