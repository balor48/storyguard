Import Guidelines

⚠️ Important

Only import files that were exported from this program. Files from other sources will likely fail to import correctly and may cause errors or data loss.

Supported File Formats

The Story Database supports importing data from the following file formats:

  • JSON files (.json) - Exported from this application

Import Process

  1. Click the "Import" button in the file controls section
  2. Choose whether to import from a file or from cloud storage
  3. If importing from a file, select the JSON file exported from this application
  4. Choose whether to import into a new database or an existing one
  5. If importing into a new database, you can provide a name or leave it blank for an auto-generated name
  6. Wait for the import process to complete

File Structure Requirements

The imported JSON file must contain the following structure:

{
  "characters": [...],  // Array of character objects
  "titles": [...],      // Array of title strings
  "seriesList": [...],  // Array of series strings
  "books": [...],       // Array of book strings
  "roles": [...],       // Array of role strings
  "customFieldTypes": [...], // Array of custom field type objects
  "relationships": [...], // Array of relationship objects
  "tags": [...],        // Array of tag objects
  "plots": [...],       // Array of plot objects
  "worldElements": [...], // Array of world building element objects
  "exportDate": "2023-01-01T00:00:00.000Z", // ISO date string
  "version": "2.0.0"    // Version string
}

ℹ️ Note

The application will attempt to handle missing properties by using default values, but the "characters" property is required for successful import.

Troubleshooting

Common Import Issues

Error Possible Cause Solution
Invalid database file format The file is not a valid JSON file or is missing required properties Make sure you're importing a file that was exported from this application
Storage Error Your browser storage may be full Export your current data as a backup, then clear some space by deleting unnecessary data
Error loading from cloud Network issues or cloud storage authentication problems Check your internet connection and make sure you're properly authenticated with cloud storage

Best Practices

  • Always create a backup of your current data before importing new data
  • Verify the imported data after import to ensure everything was imported correctly
  • If you encounter issues, try importing the file again or contact support

Need Help?

If you're experiencing issues with importing data, please check the main documentation or contact support for assistance.
